NESmaker is software for PC that allows users to create brand new, hardware playable, cartridge based games for the Nintendo Entertainment System in a modern development environment.  Users with zero coding experience can begin building their dream NES games without ever writing a line of code, while more advanced users can make use of NESmaker's development tool chain while going as far as writing entire engines from scratch.  NESmaker can help users of all skill levels create new experiences for this classic gaming system!  NESmaker games will play on any hardware based system that can play NES games, and on any NES emulator that supports mapper 30.

FREQUENTLY ASKED QUESTIONS

How does NESmaker work?

 

Games for the Nintendo Entertainment System were originally written in a programming language called 6502 Assembly.  When run through a program known as an assembler, the low-level program language is converted into hexedecimal instructions and values that are accessed directly by the NES to perform various functions.

 

Games made with NESmaker are made exactly the same way, but without the user ever having to write any code.  Under the hood are tens of thousands of lines of pre-written, interchangeable, modifiable assembly code.  When the user makes changes in the NESmaker’s intuitive graphic user interface, the underlying assembly code is updated based on those changes.  And what's great is that as you become more comfortable with NESmaker, you have full control over the code underneath and can begin writing your own to extend the tool's capabilities.

 

Choosing to export the game runs the code through the ASM6 assembler and automatically produces a NES ROM file, which opens in an emulator of your choosing (must be compatible with mapper 30).  Choosing the make-cart option exports the game and then flashes it to a mapper 30 cartridge via a Kazzo cartridge flashing device, at which time it is playable on any hardware based, NES compatible system.  To find out more about flashing hardware or flashable cartridges, please visit our website.

 

Is NESmaker easy to use?

 

NESmaker is comparable in function to other modern game development environments such as GameMaker, RPG Maker, Unity, Construct, and other similar programs.  It is designed so that beginners with zero experience can easily learn the basic workflow.  And while the quality of game you can make is still largely dependent on your creativity and understanding of good game design, NESmaker will allow you to skip much of the challenging barriers of technical understanding that have prevented many developers from creating experiences for this particilar console.

 

What kind of games can I make with NESmaker?

 

NESmaker is capable of making games of all types.  Within the NES constraints, any type of game is theoretically possible.  Keep in mind, NESmaker functions in conjunction with the underlying assembly code, much of which is interchangeable and modifiable.  What we provide are component pieces.  We give you the basic building blocks, and you determine how to put them together.  Also, as the community for NESmaker grows, so will the library of available scripts and tutorials.

 

 

LEARNING TO USE NESMAKER

The best way to learn to use NESmaker is to use the tutorials located at www.TheNew8bitHeroes.com.  They will always be current and align with the newest version.  Also, there are great communities, both on Facebook and on the NESmakers forums.

 

Below is a quick reference as to the main parts of the tool.

 

 

 

The NESmaker Interface

 

​

 

NESmaker’s main interface is made up of a standard software menu, a tool bar, and a main work area which is divided into two columns.  The column on the left is generally referred to as the project’s hierarchy. Small plus and minus symbols in the hierarchy allow you to expand or condense parts of the hierarchy tree. The hierarchy contains all of the elements that make up your current project like graphics, sounds, objects, text, and color palettes.  It also contains icons that bring up various parts of NESmaker’s tool chain, such as the pixel editor, the input editor, the map editor, and the HUD editor.

 

The larger column on the right is is the work area.  When you open an item from the NESmaker tool chain, it appears in the work area, or in some cases, as a pop up menu.

 

 

 

The Map Editor

 

​

In the map editor view, you can see your game’s screens.  Notice, there are two maps.  They care called Overworld and Underworld, but in reality, they are just two maps that hold 256 screens each.  They do not have to denote an overworld and an underworld.

 

When looking at the map editor, a map menu appears with options specific to the map screen.  Here, you can access advanced map properties and helpful map display data.

 

To enter the screen editor, simply double click on a map screen.  You can shift screens to a new map location by holding the shift key and dragging the screen to its new desire location.  You can duplicate a screen into a new location by holding control and shirt and dragging the screen to a new desired location.  If you move a screen onto an existing screen, their positions will swap.

 

Right clicking on a screen will allow you to clear the screen, access the screen info directly, and even open the screen’s latest export code in a text editor.

 

 

The Screen Editor - Tile Painter:

 

​

In the screen editor, you can create screens.  You can select a tile from the tile set and paint tiles directly to the canvas with the mouse.  You can also hold the shift key while selecting from the tileset to choose multiple tiles and with an entire tile group.

 

If your tile layout supports paths and you have selected your path group in the screen info, you can use the 1, 2, 3 and 4 keys to draw paths.  Your 6, 7, 8, and 9 key will draw your blank tile, and respectively the last three tiles in your hud tileset.  Most commonly, these can be set to solid red, green and blue so that you will be able to draw solid black as well as each of the colors in your palette without taking up tileset space for a blank tile.

 

In order to directly update attribute (color) data for a tile on the canvas, you can use the Q, W, E, and R keys while hovering the mouse over a tile.  This will change the tile’s attribute to the corresponding subpalette without having to redraw the tile or asset.

 

You can also paint with collision data.  You can choose a collision type from the collision dropdown menu, and then paint with the zero key, which will change the collision data of the tile under the mouse.  To see an overlay of collisions, you can press the collision toggle at the top of the screen.

 

Another way to paint tiles is that you can create assets.  Assets are made up of tile, collision, and attribute data all in one click.  They can also be different sizes, and be made up of multiple collision types and attributes.  For instance, you could create a house that is 3 tiles wide, with the left and right tiles being solid, but with the door in the middle being a warp.  When you place the asset on screen, the collisions will line up accordingly.

 

Since by default, the tools for the screen painter grab assets and tile groupings by the top left corner, it can often be hard to place these along the left or top of the screen if they are larger than one tile wide or tall.  When you have an asset or tile grouping selected, if you hold the shift key, you can change the handle to the right, and the option key will change the handle to the bottom.  This makes it possible to place assets of any size against the left or the top of the screen.

 

Pressing the arrow keys will load the screen in the direction of the arrow.  Pressing the tab key will toggle to the equivalent screen on the other map.

 

Right clicking on a tile will bring up a pop-up menu.  You can use this to place the player and set the warp in location.  You can also directly change the collision information for that particular tile.  Lastly, if you have monsters loaded for the screen, you can use this to place monsters, edit positions, or reset monster placements.

 

 

The Screen Editor - Tile Layout:

 

​

 

Screens can have different tileset layouts.  The standard, default layout is one main tileset, one screen specific tileset, four paths, and the HUD.  However, different games require different needs, so being able to change the tileset layout for specific needs allows for flexibility in design.  For instance, for games that do not make use of the HUD tiles, a user could make a screen with no HUD, so that they have more variety in tiles for the screen.

 

Once you have chosen your layout, you can select which tilesets to place in each slot, and those will be the tilesets available when designing this particular screen.

 

 

The Screen Editor - Screen Info:

 

​

 

 

Screens are made up of more than just graphics and collisions.  In the screen info dialog, you can set the music for the screen, choose a warp out screen, select NPC text, select the path, assign a screen type, set the object palettes, and tick screen bits for which the result is customizable from game to game.

 

 

The Screen Editor - Monster Info:

 

​

 

In the monster info tab, you can set the monsters that can be used for this particular screen and the object palettes that will be used.  In the left column, you can view what game objects will look like through the selected subpattes. There are four tabs, for four potential states that a screen can be in.

 

 

Boxes And Hud Editor:

 

​

 

In the boxes and hud editor, you can set boxes for the play area, the hud area, the textbox area, and a few other rectangles.  It is recommended that the hud and textbox area fall on a 32x32 grid to align with screen attribute values. To set a box, simply draw the box from top left to bottom right of your intended box area and click on the “SET” button for the type of box you would like that to represent.  You can also change the demo screen to see how this box layout will look lain over other map screens.

 

 

Boxes And Hud Editor - Tiles:

 

​

 

Here you can set the position in the HUD tileset that represents each character used by NESmaker’s text engine.  The demo hud tileset is already set to the default values.  If your HUD tileset differs, you can change to correct values one character at a time, or if your numbers / letters are consecutive in your tileset, you can press “set numbers” and point to 0 or “set letters” and point to A, and they will automatically set the rest.

 

 

Boxes And Hud Editor - Hud Assets:

 

​

 

Hud Assets are generally larger, multi line graphics that you would like to draw to the hud.  For instance, pictured here is a frame inside of which an item or weapon might show up.  It is one of the types of elements that can be drawn in your hud.

 

 

Boxes And Hud Editor - Hud Elements:

 

​

 

Hud elements are the contents to be drawn to the hud.  There are four types of Hud elements; Text, a number, variable images, and a hud asset.  A hud asset and text are static, while variable images and numbers are variable.  The text type draws a word or words to the HUD.  An asset is a multi line graphic that you can set up in the Hud Assets tab.  A number is the numeric value of a variable in your game.  Variable Images are effectively a health bar, where you have an empty and full image, and a variable in your game determines how many of the filled images to draw.

 

 

Asset Creator:

 

​

 

In the asset creator, you can create assets which you can then use to paint screens.  Assets contain tile, attribute, and collision data. You can create assets out of any main tileset or screen specific tileset.  You can toggle between the two by pressing the toggle switch next to the name of the tileset.  You can set a height and width of an asset, then select tiles, attributes, and collision data for each tile in an asset.

 

 

Path Manager:

 

​

 

 

In the path manager, you can set path, attribute, and collision data for your paths.  This is always set up for a four path screen, even if a screen’s tileset layout only uses two.  You can edit the path graphics by clicking on edit.  In order to be able to use this path group, you must name it and press save.  Then, if paths are party of your tilest layout when creating a screen, it will be selectable in the screen info.

 

 

Special Tile Manager:

 

​

 

 

This utility simply sets up four variables for under tiles, which are different cases available when a tile is changed.  It is mostly a holdover from Mystic Searches development, but can still be utilized by advanced users.

 

 

Palette Manager:

 

​

 

While there are many places in the tool where you can update palette information, they also have their own section.  By right clicking on a palette, you can rename it to make it easier to find in palette lists.  You can also create palette groups and use the right click button to assign a palette to a group.  When looking at a palette, you can right click on one of the color swatches to change the color for that space.  Note that change any of the left column colors will change all left column colors.  This is a quirk of how the NES handles color.  Color zero for all subpalettes will be the same, and will be transparent for sprites.

 

 

Object Manager: Animator

 

​

 

 

The Game Object manager and Monster manager are practically identical. Here you can set the width and height of an object in tiles, give it’s currently selected animation a frame count, and fill that frame with tiles.  You can also add more than one tile simultaneously by using shift and dragging in the tileset.  You can update the tile’s subpalette by clicking on the tile you want to change and then the subpalette you want to change it to.  Keep in mind, though, the color of your resulting image will be determined by the game object palettes you load to the screen.  This image represents what the object will look like if the selected subpalettes were loaded to a screen.

 

When creating animations, you can flip frames, copy and paste frames, flip entire animations, and even preview the animation with the small play button.  The slider under the play button allows you to view the animation at different speeds.

 

 

Object Manager: Animation Manager

 

​

 

Here you create names for all of your animations.  When a new animation is added to this list, it shows up in the drop down of the animator.

 

 

Object Manager: Object Info - Animations

 

​

 

In the animations tab of the Object Info, you can create animation types for this object, and then assign certain animations to the various directions.  You might have two animations, walk and stand, and 4 animations for each animation, one for each direction.  You have 8 animations in total, but only two types of animation.

 

 

 

Object Manager: Object Info - Details

 

​

 

In the details tab of the object info, you can set variables such as speed, experience worth, strength and defense.  You can also set the object type, which will affect how collisions with other objects on the game screen behave.  You can set what behaviors are called when the object runs into a solid tile or a screen edge, and you can set speed, acceleration, and jump speed.

 

 

Object Manager: Object Info - Actions

 

​

 

The actions tab is mostly for setting up a modular AI system for monster type objects, but is also used to tie animation types to action states.  Each object can have eight action steps.  Each action step can be tied to an animation type.  Drop down menus determine the type of behavior taken by the object in that step, and what occurs at the end of the animation or when an action timer goes off.  There is also an animation slider which controls the speed of the animation for that frame.

 

You can also use and customize the action step flags for purposes specific to your game.  For instance, perhaps a monster is invincible during one particular type of action.  You could make a flag for invincibility and tick it for that action.

 

 

Object Manager: Object Info - Bounding Box

 

​

 

 

To create a bounding box for a player, simply click and drag across the player.  This is the bounding box the object will use for both tile collisions and collisions with other objects.  You can also adjust the left, top, width or height of the bounding box manually.

 

 

Monster Groups:

 

​

 

To add monsters to your screens, you’ll have to create monster groups.  Groups can contain four different monsters.  All monsters in a group must be from the same tileset.  Choose a tileset, then choose the monster type from the dropdown menu.  Give the monster group a name and press save.  Now this monster group can be used in the screen info to be able to add these types of monsters to a game screen.

 

 

Text String Editor:

 

​

 

In the text string editor, you can create all of the text strings for your game.  You can give them a name.  You can also set end of text action.  For instance, perhaps you want to trigger an event or warp to a new screen or give the player a power up.  The actions listed are default examples, and may need some light coding to work appropriately for different games, and the data can be utilized to do anything that might involve reading and evaluating a hex value.

 

 

Text String Group Editor:

 

​

 

When you add NPC text to a room, you are adding a text group, which is a collection of four strings.  The monster placed at monster placement 1 will read the first string, the monster placed at monster placement 2 will read the second, and so on.  You can name text groups for easy identification from the screen info dialog box.

 

 

Input Scripts:

 

​

 

Input scripts are a very particular case, as they work with controller reads.  To add input scripts to your game, click on input scripts and navigate to the input script you want to add.  Double click on it in the file browser and it will disappear from the browser and add to your input scripts.  It is still in its original location, but since you already have it loaded, this prevents you from loading the same script multiple times.

 

 

Defined Scripts:

 

​

 

Defined scripts are all of the scripts that have definitions for you module.  You can also view and set these in the scripts settings tab in the project settings.  When you click on defined scripts, it brings up an internal script editor.  You can edit scripts here.  You can also see your included list of macro functions in the rightmost column.  Clicking on these will bring up their definition at the bottom of the screen in case you need a quick reminder of what they do or how to use them.  Double clicking a macro will add it in line into your code.

 

 

Input Handler:

 

​

 

The input handler is how to attach input scripts to controller input.  From the input handler, you can choose the game state where a particular read should take place, an target object to receive the input, the controller that should be read, the input that you want to call the script, the button state of that input (either pressed, held, or released), and marry it to a script by choosing all of the above options and a script from the Script To Run drop down menu, then pressing add script.

 

 

Pixel Editor:

 

​

 

The pixel editor allows you to create graphics that work within NES’s graphic constraints, either by drawing them directly with common drawing tools, or by importing graphics and conforming them.  You can see the graphics in rgb+a format, which correlates to the four colors in each subpalette. Alpha represents the first subpalette color, red represents the second, green represents the third, and blue represents the fourth.  NESmaker works with these pure value bmp files and translates through the select NES palette as a reference for white the tileset will look like in game.

 

 

Project Settings:

 

​

 

In the project settings, a user can do many things.   Project labels can be created, which are exposed to the various GUI interfaces.  For instance, a user can set a custom name for collision types, or give in-tool names to the tilesets.  A user can create custom variables with initial values.  A user can define and link scripts, which is the heart of how every game functions.  A user can set up export settings in order to create a project bundle which includes relevant artwork and the ROM file.
